FPGA 笔记

FPGA

VHDL是一门很吊的语言。从书上得到的信息来看，这个软件我们自己都还没有摸清楚？但它强大的设计电路功能让人眼前一亮。

个人觉得VHDL这门语言与以往所学的语言大相径庭，但也有共同之处，你甚至能在里面看到很多汇编的影子。学一门语言我是打算按程序结构入手，逐渐递进，最后到完整的程序设定。

VHDL程序结构

每一种语言都需要固定的格式规范，在C系列中就是标志的Using namespace std，Java里面的Public class void main，Python严格的缩进制度…这些规范成就了每一门语言的功能与针对性。同样的，作为一种电路设计语言，VHDL的程序分成如下五个部分：

  • 设计库-Library

        存放已编译的实体、结构体、包集合和配置

    ​ Storage the entity,Architecture,Package and configuration that have been complied

  • 包集合-Package

        存放共享的数据类型、常数和子程序等

    ​ Storage the constant,shared data type and sub program

  • 实体-Entity：

        描述所设计系统的外部接口信号

    ​ Describe the outer signal the system you design

  • 结构体-Architecture

        描述系统内部的结构与行为

    ​ Describe inner structure and behavior of the system

  • 配置-Configuration

        从库中选取所需单元组成设计的不同结构体

    ​ Select the units u need to construct different Architecture

Entity

  • 语法形式：
entity 实体名is
        [generic (类属参数说明) ;] --这行其实我们实际应用很少有，哦对VHDL是--作为注释
        [port (端口说明) ;]
    end [entity] [实体名]；

端口里面也有分类：

  • In （输入）
  • Out（输出）
  • Inout （双向）
  • Buffer （输出，但这个输出可以在内部再次利用）
  • linkage（无指定方向）

In 信号只能被引用,不能被赋值；不可以出现在<= 或 : = 的左边

In signal can only be quoted but no equalize. It can’t be shown on the left of “<=” or”:=”

out 信号只能被赋值,不能被引用；不可以出现在<= 或 : = 的右边

Out signal can only be equalized bu no be quoted. It can’t be shown on the right of “<=” or “:=”

buffer 信号可以被引用,也可以被赋值；可以出现在<= 或 : = 的两边

Buffer signal can also be quoted and equalized. Both side can it put of “<=” or “:=”

Architecture

  • 语法形式：
architecture 结构体名of 实体名is
    [说明语句] 内部信号、常数、元件、数据类型、函数等
begin
    [并行处理语句和block、process、function、procedure]
end [architecture][结构体名]
  • 结构体描述一个设计的结构或行为，把一个设计的输入和输出之间的关系建立起来
    一个实体可以有多个结构体，分别代表该实体的不同实现方案，各个结构体的地位是同等的。但很可惜的是，我们遇到的程序基本上是单个结构体的。

而Architecture种也有子结构：

好家伙，你发现这玩意完全就可以替代掉Architecture，那为什么还要大费周折高出已给Block呢？

因为Block的出现，是为了改善并行语句和结构的可读性分块转换变量为局部变量，从而保护某些信号。

  • 并发执行，与描述顺序无关。
  • 语法形式 ：
块名称：BLOCK
          [接口说明；]
          [说明部分；]
         BEGIN
           并行处理语句；
         END  BLOCK 块名称；

所有在Block里面定义、说明的信号，在Block外面来看是无法访问的。

Process 进程

Process存在的意义又是什么呢？Process其实完成了我们接触过的大部分顺序结构的工作：顺序执行程序指令。你可能会很惊讶，我们习以为常的顺序编程为什么在VHDL中被“特殊化“了，我个人的理解是对于一个电路来讲，它存在的意义是执行给定的指令，而执行效率最高的方式无疑是多任务多开的并行模式，所以不出意外我们都希望它是并行执行程序。一个Process对敏感信号保持持续扫描状态，一旦某一个敏感信号满足条件马上激活进程，完成对应的指令。

但请注意，对于某些比较烦人的单变量输出变化的要求，不得已我们才能使用顺序结构的语句来描述器件行为。虽然Process是顺序执行的，但在一个结构体中，可以有多个Process并行运算，你懂我意思吗？举个例子，你在鹏翔公寓的洗漱房，每天早上你拖着你懒惰的身子来到一个水龙头前，旁边也有一样不愿早起的懒狗。你的洗漱顺序是先刷牙再洗脸，但你旁边那个小帅可能是先刷牙然后刮个胡子，最后再洗脸。你们的”进程“是不同的，但水龙头流出来的水是独立的，你们互不影响。

  • 顺序执行，与描述顺序有关。
  • 语法形式 
进程名称：PROCESS（敏感信号表）[IS]
          [接口说明；]
          [说明部分；]
         BEGIN
           顺序处理语句；
         END  PROCESS 进程名称；

敏感信号其实未必列在信号表内，它可以是WAIT UNTIL clock一类的说明语句来激活进程。

Subprogram 子程序

subprogram也分为两种：Procedure和Function。

  • 使用顺序语句。
  • 目的：完成各种不同的定义和算法描述。
  • 特点：完成重复性的计算工作，通常放置在包集合当中，被不同的实体所调用。可重载的特性，即允许有多个重名的子程序存在（具有不同的参数类型或者返回值类型）。

我知道肯定也会有人问，他妈的都一样是顺序结构，为什么Process还不够用？还来个Procedure？当我们英语很好分得清吗？

Procedure和Function的出现是为了多次利用。对于Process而言，他的返回值虽然没有明说，但我们动用一下小脑袋晃晃里面的水就不难发现：

Process的状态只有两个：等待执行和执行中。如此的局限性，自然不能对多个对象进行多次重复的操作，所以Function和Procedure就诞生了，这里我们可以看到i作为高级语言应有的尊严：至少他是有面向对象的意识了！不像汇编那么落智了！

Function 函数：

Function有一种微缩Entity+Architecture的思想在里面，他需要函数段和函数体两部分来构成。像不像C++？

  • 语法形式 
FUNCTION 函数名（参数表） 
    RETURN 数据类型 ；
    语法形式 
FUNCTION 函数名（参数表） 
    RETURN 数据类型 IS
          [定义语句]
    BEGIN
           顺序处理语句；
            RETURN 返回值
END  函数名称；

  • 用于对函数体的说明，放置在包集合的说明部分，对应于包集合内的函数体。不是必需的，在结构体中定义函数时，不需要函数首的说明。

  • 重载。重载就是参数不同的同名函数的应用，值得注意的是四则运算（+-*/）的重载需要加双引号。

Procedure 过程：

Procedure和Function一样由首+体构成。但很好玩的一点就是，在进程（Process）或结构体（Architecture）中不必定义过程首，而程序包中则必须定义。

  • 语法形式 
PROCEDURE 过程名（参数表） IS
          [定义语句]
        BEGIN
           顺序处理语句；
         END  过程名称；
  • Procedure和函数一样可以重载。

Procedure 和Function的联系、区别在于：

  • 函数可以有输入参数/也可以没有输入参数，必须有一个返回值（return）。
  • 过程必须有参数，这些参数必须有数据类型（输入/输出/双向），对于输入类型默认为常量，对于输出/双向类型默认为变量，无return。
  • 函数调用可作为表达式的一部分，过程则直接调用。

Package

请留意我的分级，我把Package放到Architecture和Entity规划到同一类级别。

我个人对Package的理解是面向对象编程中的”类“。这个类里面可以定义很多骚操作/骚函数。

  • 罗列VHDL语言中所要用到的信号定义、常数定义、数据类型、元件描述、函数定义以及过程定义等内容。

  • 包集合的使用

    语法形式：
  USE [库名].[包集合名].[项目名]

    如：

     LIBRARY IEEE
USE IEEE.STD_LOGIC_1164.ALL

  • 语法形式：

    PACKAGE [包集合名] IS
     [说明语句]
   END [包集合名] 
    PACKAGE BODY [包集合名] IS
     [说明语句]
   END [包集合名]

Package abc is
    function str_to_int(S : string) return integer;
End abc

Package body abc is
   function str_to_int(S : string) return integer is
   begin
   ………..
   end str_to_int; 
End abc

到这里，你已经对VHDL这门语言有一定的初步了解了。你现在应该能做到：

  • 拿到程序后，知道程序的执行顺序
  • 拿到程序后，分辨出各个变量类型
  • 拿到程序后，不至于无话可说

VHDL操作对象类型

很有趣的一点就是，VHDL中的操作对象类型很少，只有三种：Constant，Variable和Signal。

但类型少并不是它有趣的理由，有趣的理由在于，作为一门对类型极度严格要求的语言，在这三种类型衍生出来的数据类型却五花八门：

数组，整数……很多很多。这从另一个维度说明了VHDL的大道至简的哲学思想。

  • 常数(constant):是一个固定的值，常数说明就是对某一常数名赋予一个固定的值，常量只能在说明时被赋值。
    语法形式:constant 常数名：数据类型：=表达式;
  例: constant VCC ：real ：= 5.0;
  • 变量(variable):只能在process、function和procedure结构中使用，是一个局部量。
    语法形式:variable 变量名:数据类型 约束条件:=表达式;
  例:variable x,y ： integer;
      variable count ： integer range 0 to 255 ：= 10;
  • 信号(signal):电路内部硬件连接的抽象。信号可以在architecture、package、entitiy中声明。是一个全局量。
    语法形式:signal 信号名:数据类型 约束条件:=表达式;
  例:signal clk ：std_logic ：= ‘1’;

信号和变量，一个针对全局，一个是局部。但二者的使用，又有什么不同呢？

  • (1) 信号赋值是有一定延迟的 ，在进程中，信号的代入是在 
     进程结束或者WAIT语句时进行。 而变量赋值没有延迟。
  • (2) 对于进程语句来说，进程只对信号敏感 ，不对变量敏感。
  • (3) 信号在某一时刻除了具有当前值外 ，还具有一定的历史
     信息 ( 保存在预定义属性中 ) ，可以形成波形 。而变量
 在某一时刻仅包含一个值。
  • (4) 信号可以是多个进程的全局信号 ，实现在多个进程之间
     传递信息; 而变量只在定义它的过程、函数和进程中可见。
  • (5) 信号是硬件中连线的抽象描述，其功能是保存变化的数
     据值和连接子元件。变量在硬件中没有类似的对应关系，
 主要应用于高层次的建模中。
  • (6) 信号赋值和变量赋值的格式如下所示 :
              信号 < =表达式 ;
    变量 ：=表达式 ;

VHDL预定义数据类型

  • 布尔数据类型(只有FALSE和TRUE两种取值)

  • BIT数据类型(BIT_VECTOR数据类型‘0’、‘1’取值)

  • 字符数据类型（字符串数据类型‘A’）

  • 整数数据类型（自然数和正整数数据类型，注意他的取值范围在-2147483647~+2147483647）

    表示进制的时候加上#，比如16#D#表示十六进制整数，2#11001100#二进制整数，10E4十进制整数

  • 实数数据类型（仅能在仿真器使用）

  • 时间数据类型（唯一的物理量纲，不能在仿真器中使用）
