CPU模拟

发表于 2023-12-12 更新于 2024-01-08 分类于 杂项

本章的主要目标是将我们的第一个NES游戏跑起来。我们将要游玩贪吃蛇，源代码可以在这里找到。

CPU是任何计算机系统的心脏。CPU的工作是运行程序指令，协调所有可用的硬件模块，以提供完整的体验。尽管PPU和APU运行着各自独立的电路，但它们仍然要在CPU的节拍下行进，执行CPU发出的命令。

在我们跳到实现部分的工作前，我们需要简要地讨论下哪些资源是CPU用于完成工作的可用资源。只有两种类型：内存映射和CPU寄存器。从编程的角度来看，内存映射只是一个1字节单元的连续数组。NES CPU使用16位内存寻址，这意味着它可以寻址65536（$2^6$）个不同的内存单元。

正如我们之前所见，NES平台只有和CPU直连的2KiB的RAM：

  • RAM可以通过 [0x0000 … 0x2000]的地址空间来访问。

  • [0x2000…0x4020]将会重导向导其他可用的NES平台的硬件模块：PPU,APU,游戏手柄部分之类的。

  • [0x4020 .. 0x6000]是一块特殊的空间，不同世代的卡带将会根据世代的不同占用不同的部分。它可以被映射到RAM，ROM或者什么都没做。这一块空间是被称作“mapper”的东西——卡带上的特殊电路来控制的。我们会忽略这部分的内容。
  • [0x6000 .. 0x8000] 是卡带的RAM的保留空间，如果卡带有RAM的话。他曾经被用于像塞尔达一类的游戏来存储和检索游戏的状态。我们也会忽略这部分的内容。
  • [0x8000 … 0x10000] 的空间将会映射到卡带的PRG ROM。

内存访问相对较慢，NES的CPU有一些称为寄存器的内部内存插槽，访问延迟明显较低。

CPU 操作类型 执行时间（CPU周期）
寄存器寻址 2
RAM前255字节寻址 3
RAM255字节后寻址 4-7

NES有六个寄存器：

  • Program Counter (PC) - 程序计数器，保存下一条执行机器语言指令的地址。
  • Stack Pointer - 堆栈指针，内存空间中的[0x0100…0x1FF]是用于堆栈的构建。 堆栈指针保存了该空间顶部的地址。NES堆栈(和所有堆栈一样)从上到下增长：当一个字节被压入堆栈时，SP寄存器递减。当从堆栈中检索到一个字节时，SP寄存器递增。
  • 累加寄存器 (A) - 用于存放逻辑、计算和内存访问的操作结果。它用于某些操作的输入参数。
  • 索引寄存器X (X) -在特定的存储器寻址模式中用作偏移量(稍后将详细介绍)。可用于辅助存储需求(保存温度值，用作计数器等。)
  • 索引寄存器Y (Y) - 功能类似索引寄存器X。
  • 处理器状态(P) - 8位寄存器代表7个状态标志，可以根据最后执行的指令的结果设置或取消设置(例如，如果操作结果为0，则设置Z标志1，否则取消设置/擦除0)。

每个CPU都有一个预定义的硬连线指令集，它定义了CPU可以做的一切。CPU从应用层接收机器代码形式的指令。你可以把机器语言想象成连接软件和硬件的薄层。6502芯片的完整指令集可以在下面查到：

  • https://www.nesdev.org/obelisk-6502-guide/reference.html
  • http://www.6502.org/tutorials/6502opcodes.html

我倾向于使用这两个链接。这些页面提供了可用CPU特性的完整规格及其机器代码。在我们继续前，我强烈建议阅读 interactive tutorial on 6502 instructions这篇文章。

6502芯片是比较简单的CPU它只支持六种类型的命令和大约64种独特的命令。因为一些指令对于不同的存储器寻址模式有多个版本，这导致我们要实现大约150个机器代码操作。

注意：NES控制台使用的是基于6503的定制芯片2A03，但他们有很大的差别：

  • 除了官方的机器指令之外，它还有大约110个非官方的附加操作码(幸运的是，其中大约三分之一是空操作)
  • 拥有板载的APU模块
  • 不支持十进制算数

为了简单起见，我们需要实现对256种不同机器指令的支持。好消息是指令之间有很多相似之处。一旦我们有了合适的基础，我们将不断地重用它们来实现整个集合。
